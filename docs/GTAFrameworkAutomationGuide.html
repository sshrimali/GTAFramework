Guide to Writing tests

Test Suites

Tests are organized in following structure of packages & classes

Test Suite Class

src/<FeatureName>/</TestSuiteName>.java

Test Suite Sample

src/testsuites/Authentication/SignInTest.java

Test Case

Testcases are written as simple java methods using '@Test' annotation of testNG.

Testcase Syntax:

@Test public void test-description_testcaseId() throws Exception { <UtilityHelper> object = new <UtilityHelper> (dataProperty,"testcaseID"); object.method(); }

Testcase Sample:

@Test public void signUpFrontend_1145() throws Exception { AuthenticationHelper helper = new AuthenticationHelper(dataProperty,"1145"); helper.signup(); }

Test Data

Test data is equivalent of property files where key value pairs are stored.

Test Data Location:

Location: <testdata>/<functionality>/</featurename.txt> Sample: testdata/authentication/signup.txt

Test Data Syntax:

<testcaseId>_parameter1=value1 <testcaseId>_parameter1=value1 ....
